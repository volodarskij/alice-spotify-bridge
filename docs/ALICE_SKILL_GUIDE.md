# Гайд: свой навык Алисы для управления чем угодно

Пошаговая инструкция, как сделать приватный голосовой навык Яндекс.Алисы с вебхуком на своём сервере — на примере навыка управления Spotify из этого репозитория. 90% гайда применимо к любому самодельному навыку (умный дом, свой медиасервер, что угодно с HTTP API).

---

## Как это работает

```
«Алиса, попроси <имя навыка> включить Coldplay»
        ↓
Яндекс.Диалоги (облако)
        ↓  POST https://ваш-домен:8889/alice  (JSON)
Ваш webhook-сервер
        ↓  JSON-ответ (text + tts) за ≤ 3 секунды
Алиса произносит ответ
```

Навык — это просто HTTPS-эндпоинт, который принимает JSON с распознанной фразой и возвращает JSON с текстом ответа. Всё остальное (распознавание речи, синтез голоса) делает Яндекс.

## Шаг 1. Домен и HTTPS

Вебхук обязан быть на **HTTPS с валидным сертификатом** (самоподписанный не подойдёт). Для домашнего сервера:

1. **Домен**: любой DDNS-сервис (DuckDNS, ClouDNS и т.п.) или свой домен, A-запись на ваш белый IP.
2. **Проброс порта**: на роутере пробросьте внешний порт (например 8889) на хост с навыком. Пробрасывайте **только** порт вебхука.
3. **Сертификат**: Let's Encrypt через `certbot` или `acme.sh`:
   ```bash
   acme.sh --issue -d ваш-домен.example.com --standalone
   ```
   Полученные `fullchain` и `key` укажите серверу навыка (`SSL_CERT_FILE` / `SSL_KEY_FILE` в нашем случае). Не забудьте про автопродление (сертификат живёт 90 дней).

Если белого IP нет — подойдёт туннель (Cloudflare Tunnel и аналоги), лишь бы снаружи был валидный HTTPS.

## Шаг 2. Webhook-сервер

Минимальный контракт Диалогов:

**Запрос** (Яндекс → вы), сокращённо:
```json
{
  "session": { "new": true, "session_id": "...", "user_id": "..." },
  "request": { "command": "включи coldplay", "original_utterance": "Включи Coldplay" },
  "version": "1.0"
}
```

**Ответ** (вы → Яндекс):
```json
{
  "response": {
    "text": "Включаю Coldplay.",
    "tts": "Включаю Coldplay.",
    "end_session": false
  },
  "version": "1.0"
}
```

Ключевые правила:

- **Таймаут ~3 секунды.** Не успели — Алиса скажет «навык не отвечает». Всё медленное (поиск, API-вызовы) либо укладывайте в лимит, либо отвечайте сразу и доделывайте асинхронно. В нашем навыке LLM-парсинг ограничен `--max-time 1.5`, а при просрочке срабатывает regex-фолбэк.
- **`session.new: true` + пустой `command`** — пользователь просто открыл навык («Алиса, запусти Спотик»). Ответьте приветствием с `end_session: false`, чтобы остаться в диалоге.
- **`end_session: true`** завершает диалог (наш навык делает так по «выход», «хватит»).
- В `request.command` приходит нормализованный текст в нижнем регистре, в `original_utterance` — как сказал пользователь.
- Отвечайте HTTP 200 даже на внутренние ошибки — верните `text: "Произошла ошибка"`, иначе пользователь услышит стандартную заглушку.

Полный формат: [документация Яндекс.Диалогов](https://yandex.ru/dev/dialogs/alice/doc/ru/request).

## Шаг 3. Регистрация навыка

1. Откройте [dialogs.yandex.ru/developer](https://dialogs.yandex.ru/developer) → **Создать диалог** → **Навык в Алисе**.
2. Заполните:
   - **Имя навыка** — как Алиса будет его называть.
   - **Активационные имена** — фразы вызова. Выбирайте то, что Алиса стабильно распознаёт: 2–3 слога, без англицизмов, которые ASR коверкает. Мы используем «Спотик» — «Спотифай» Алиса часто слышит как «спорт и фай».
   - **Backend** → **Webhook URL**: `https://ваш-домен:8889/alice`.
   - **Доступ** — «Приватный»: навык видите только вы, модерацию проходить не нужно.
3. Сохраните. Во вкладке **Тестирование** есть чат-эмулятор — можно дебажить, не трогая Станцию (там же виден сырой JSON запросов и ответов).
4. **Черновик уже работает на ваших устройствах**: если Станция привязана к тому же аккаунту Яндекса, скажите «Алиса, запусти <имя навыка>» — публиковать ничего не нужно.

## Шаг 4. Парсинг команд

Внутри навыка фраза «включи горгород оксимирона» должна превратиться в структурированный запрос. Наш подход двухуровневый:

1. **LLM** (Groq, `llama-3.1-8b-instant`, бесплатный тир): system-промпт просит вернуть JSON `{type, query, artist, album}`, транслитерировать русские написания артистов («битлз» → The Beatles) и строить точные поисковые запросы `artist:X album:Y`. Ответ за ~400 мс.
2. **Regex-фолбэк**: если LLM не ответил за 1.5 с или ключа нет — набор регулярок вырезает глаголы («включи», «поставь»), слова-маркеры («альбом», «плейлист») и мусорные слова. Важно: `\b` в JS **не работает с кириллицей**, используйте `(?:^|\s)`.

Такая схема даёт умный парсинг, но навык не умирает без LLM.

## Шаг 5. Типовые грабли

| Проблема | Решение |
|----------|---------|
| «Навык не отвечает» | Ответ дольше 3 с. Профилируйте: у нас на каждый запрос пишется таймер в структурированный лог |
| Алиса не понимает имя навыка | Смените активационное имя на более простое фонетически |
| Вебхук не вызывается | Проверьте сертификат: `curl -v https://домен:порт/alice` не должен ругаться на TLS |
| Навык вылетает из диалога | Вы вернули `end_session: true` или забыли поле `version` |
| Кириллица ломает shell-команды | Не интерполируйте текст в командную строку; передавайте тело через файл (`--data-binary @file`) и секреты через env |

## Специфика этого проекта: связка со Spotify

### Spotify-приложение

1. [developer.spotify.com/dashboard](https://developer.spotify.com/dashboard) → Create app.
2. Redirect URI: `https://ваш-домен:8889/callback`.
3. `Client ID` и `Client Secret` → в переменные окружения (`.env.example` в корне `alice-skill/`).

### Одноразовая OAuth-авторизация

```bash
export SPOTIFY_CLIENT_ID=... SPOTIFY_CLIENT_SECRET=... SPOTIFY_REDIRECT_URI=https://ваш-домен:8889/callback
node spotify_auth.js
```

Скрипт напечатает URL — откройте его, разрешите доступ, и `refresh_token` сохранится в `spotify_tokens.json`. Дальше навык сам обновляет access token (Spotify иногда ротирует refresh token — это тоже обрабатывается).

### Session_id

Мосту нужна кука `Session_id` Яндекса, чтобы получать краткоживущие Glagol-токены (`https://quasar.yandex.ru/glagol/token`):

1. Залогиньтесь на [yandex.ru](https://yandex.ru) в браузере.
2. DevTools (F12) → Application/Storage → Cookies → `https://yandex.ru` → скопируйте значение `Session_id`.
3. Передайте мосту через `--session-id`.

⚠️ Эта кука — полный доступ к вашему аккаунту Яндекса. Не коммитьте её, не вставляйте в issues и логи. Когда протухнет (обычно через несколько месяцев), мост вернёт 401 и пометит `session_id_valid: false` в `/health` — повторите процедуру.

### Что умеет навык голосом

- «включи *трек/артиста*», «включи альбом *X* от *Y*», «включи мой плейлист *Z*»
- «пауза» / «стоп», «продолжи», «дальше», «назад»
- «что сейчас играет»
- «выход» — закрыть навык
